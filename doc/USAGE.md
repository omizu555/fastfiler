## 1. はじめに

### 1.1 起動方法

リリースビルドを起動すると、初回はホームフォルダ (`C:\Users\<あなた>`) を 1 タブだけ開いた状態で立ち上がります。
2 回目以降は前回のセッション (タブ構成・パス・選択・スクロール位置・分割比率・パネル配置・ホットキー) を自動で復元します。

二重起動防止が入っており、すでに起動中のときは既存ウィンドウがフォアグラウンドに来ます。

開発モードでの起動は [付録 A. ビルド](#付録-a-ビルド) を参照。

### 1.2 アンインストール / リセット

すべての設定は以下に保存されています。

| 種類 | 場所 |
|---|---|
| セッション / 設定 / ホットキー | ブラウザ localStorage (`fastfiler:state:v1`) |
| プラグイン本体 | `%APPDATA%\fastfiler\plugins\<id>\` |
| プラグイン KV ストア | `%APPDATA%\fastfiler\plugins\<id>\storage.json` |

リセットしたい場合はアプリを終了して該当ファイルを削除するか、DevTools コンソールで `localStorage.clear()` を実行してください。

## 2. 画面構成と用語

### 2.1 全体レイアウト

```
┌───────────────────────────────────────────────────────────┐
│ タイトルバー (Windows 標準)                                │
├──────────┬────────────────────────────────────────────────┤
│          │  パンくず ↑ ✎ … (ペインツールバー)              │
│ 縦タブ   ├────────────────────────────────────────────────┤
│ + ドック │                                                │
│ レイアウト│   メインペイン (ファイル一覧 / ツリー)         │
│          │                                                │
│          ├────────────────────────────────────────────────┤
│          │  ターミナル (任意)                             │
├──────────┴────────────────────────────────────────────────┤
│ ⚡FastFiler                              ⌨ ⚙   ←ステータスバ│
└───────────────────────────────────────────────────────────┘
```

ヘッダ (上部ツールバー) は v3.6 で廃止され、操作系は **ボトムステータスバー** に集約されています。
画面領域を最大化するため、表示切替系は基本ホットキーで行う設計です。

### 2.2 用語の整理

| 名称 | 場所 | 説明 |
|------|------|------|
| タブパネル | 左 / 右 / 上 / 下 にドック可 | 縦タブと「＋新規」ボタン |
| ワークスペースツリー | 同上 | ドライブ起点のグローバルツリー (パネル全体を 1 本のツリーで操作) |
| メインペイン | 中央 | 分割可能なファイル一覧領域。各タブが独自の分割構成を持つ |
| ペイン内ツリー | 各メインペイン内 | 📋/🌲 切替で「リスト」「ツリー」を選べる、列挙対象パスのツリー |
| プレビュー | 右 | 選択ファイルのプレビュー (`Ctrl+P`) |
| プラグインパネル | 右 | プラグイン UI (`Ctrl+Shift+P`) |
| ターミナル | 下 | 内蔵 PowerShell / cmd など (`Ctrl+\`) |
| ステータスバー | 最下部 | 左にロゴ、右に ⌨ ⚙ |

### 2.3 ステータスバーの操作

| ボタン | 機能 |
|---|---|
| `⌨` | ターミナルパネルの表示 / 非表示 (アクティブ時はハイライト) |
| `⚙` | 設定ダイアログを開く |

そのほかのパネル切替 (タブ / ツリー / プレビュー / プラグイン) は **ホットキー** から呼び出します ([13. ホットキー一覧](#13-ホットキー一覧))。

#### 通知 / ジョブ進捗 (v1.9〜)

- **完了通知** はステータスバー左寄りに 1 行で短時間表示されます (旧: 右下ポップアップは廃止)。
- **進行中のコピー / 移動 / 削除** はステータスバー内に進捗バー + 件数 / バイト数 + `✕` (キャンセル) で表示されます。複数同時進行時は最新 1 件 + `他 N 件` バッジ。

#### 終了時の状態保存 (v1.9〜)

- アプリ終了時に開いていた **タブ / ペイン構成 / 各ペインの表示パス** は自動保存され、次回起動時に復元されます (`localStorage` 永続化 + 終了直前の即時 flush)。

## 4. ファイル / フォルダ操作

### 4.1 基本操作 (移動 / 選択)

| 操作 | キー / マウス |
|---|---|
| 開く / 入る | ダブルクリック / `Enter` |
| 親フォルダへ | `Backspace` または ペインツールバー `↑` |
| 再読込 | `F5` |
| 全選択 | `Ctrl+A` |
| 範囲選択 | `Shift+クリック` / `Shift+↑/↓` |
| 追加選択 | `Ctrl+クリック` |
| 矢印で移動 | `↑/↓/←/→` |

ファイルリストは **行高 28px の仮想スクロール** で動作するので、`C:\Windows\System32` のような数万件フォルダでも瞬時に開きます。

### 4.2 コピー / 切り取り / 貼り付け / リネーム / 削除

| 操作 | キー |
|---|---|
| コピー | `Ctrl+C` |
| 切り取り | `Ctrl+X` |
| 貼り付け | `Ctrl+V` |
| 名前の変更 | `F2` |
| 新規フォルダ | `Ctrl+Shift+N` |
| 新規ファイル | コンテキストメニュー → **新規ファイル** サブメニュー (v1.11) |
| ゴミ箱へ送る | `Delete` |
| 完全削除 (戻せません) | `Shift+Delete` |

`Delete` は OS のゴミ箱経由なので誤操作時もリカバリ可。`Shift+Delete` は警告ダイアログを経て即時削除します。

#### 4.2.1 新規ファイル (v1.11)

ペイン上で右クリック → **新規ファイル** から、内蔵テンプレートまたはユーザー定義テンプレートを基にファイルを作成できます。

**内蔵テンプレート (13 種)**
- テキスト/Markdown: `空のテキスト (.txt)` / `Markdown (.md)`
- データ: `JSON (.json)` / `YAML (.yaml)` / `CSV (.csv)` / `XML (.xml)` / `INI (.ini)` / `TOML (.toml)`
- スクリプト: `Bat (.bat)` / `PowerShell (.ps1)` / `Shell (.sh)`
- Web/コード: `HTML (.html)` / `JavaScript (.js)`

作成すると同名ファイルがあれば自動で `(2)`, `(3)` … が付与され、その場で名前の変更ダイアログが開きます。`Esc` でキャンセル可、`Ctrl+Z` で取り消せます。

**ユーザー定義テンプレートの追加方法**

`%APPDATA%\fastfiler\templates` フォルダ (= 設定タブの **「テンプレ フォルダを開く」** ボタンで開けます) に任意のファイルを置いてください。次回起動時、または設定タブの **「🔄 再読込」** ボタン押下で「新規ファイル」サブメニューに表示されます。

例: `%APPDATA%\fastfiler\templates\契約書ひな型.docx` を置くと、サブメニューに「契約書ひな型.docx」が出現し、選択するとカレントペインへコピーされます。

> **💡 エクスプローラとのクリップボード連携**
> `Ctrl+C` / `Ctrl+X` を実行すると Windows クリップボードにもファイルパスが書き込まれます。
> FastFiler でコピー / 切り取りした後、エクスプローラや他のアプリで **`Ctrl+V` して貼り付け** ることができます。
> 切り取り (`Ctrl+X`) の場合、エクスプローラが貼り付け完了時に元ファイルを自動削除します。

### 4.3 ドラッグ & ドロップ (左ボタン)

#### 内部 D&D (FastFiler 内ペイン同士)

ペインの行を左ボタンで掴み、別ペインまたは同ペインのフォルダ行へドロップします。

| 状況 | 既定動作 | `Ctrl` 押下時 |
|---|---|---|
| 同一ドライブ間 | **移動** | コピー |
| 別ドライブ間 | **コピー** | コピー |

ドロップ先の解決優先順は **フォルダ行 → パンくずセグメント → ペイン背景 (現在フォルダ)** です。

> ※ 内部 D&D は OS の HTML5 ドラッグではなく、専用の pointer エンジンで実装しています。
> Windows の `dragDropEnabled` 環境では HTML5 ドラッグが自プロセス内に届かないための対応です。

#### 外部 D&D (エクスプローラ → FastFiler)

エクスプローラや他のアプリからファイルをドラッグし、FastFiler のペインまたはフォルダ行にドロップできます。

| 状況 | 既定動作 | `Ctrl` 押下時 |
|---|---|---|
| 同一ドライブ | **移動** | コピー |
| 別ドライブ | **コピー** | コピー |

#### FastFiler → 外部アプリ (OS ドラッグアウト)

| 方法 | 操作 |
|---|---|
| **クリップボード経由** | FastFiler で `Ctrl+C` / `Ctrl+X` → エクスプローラ等で `Ctrl+V` |
| **OS ドラッグ** | 5px 以上ドラッグ → エクスプローラ等へドロップ |

> `Shift+クリック` (動かさない) は従来通り**範囲選択**です。閾値を超えたドラッグ移動が始まった時点で初めて OS ドラッグが起動します。

#### 自身フォルダへのドロップ防止

ドラッグ元と同じフォルダ (=現在ペインの背景) へのドロップは **無視** されます (無限ループ・無駄な再描画を避けるため)。同階層でフォルダを掴んでフォルダ内に落とす操作は通常通り動作します。


### 4.4 右ボタン D&D メニュー

ファイルを **右ボタンで掴んでドロップ** すると、ドロップ位置にメニューが表示されます。

| 項目 | 動作 |
|---|---|
| 📁 ここに移動 | 移動 |
| 📄 ここにコピー | コピー |
| ✖ キャンセル | 何もしない |

`Esc` でも常時キャンセルできます。Windows 標準と同じ操作感です。

### 4.5 コンテキストメニュー / プロパティ

ファイル / フォルダを **右クリック** で表示。

- 開く / 親フォルダ / 切り取り / コピー / 貼り付け / 名前の変更 / 削除
- **新規ファイル** サブメニュー (v1.11) ── 内蔵「空のファイル」+ ユーザー定義テンプレート
- **ユーザー コマンド** (v1.13) ── `commands.json` で定義した外部コマンド (詳細は §4.6)
- **エクスプローラで表示** ── Windows 標準のエクスプローラを起動
- **プロパティ** ── サイズ・更新日時・属性を表示 (属性ダイアログ)
- プラグインが登録した追加項目 (アクティブなプラグインのみ)

#### 4.5.1 .iso / .img / .vhd / .vhdx をダブルクリック (v1.11)

ディスクイメージ (`.iso` / `.img` / `.vhd` / `.vhdx`) をダブルクリックすると、Windows エクスプローラと同じ **「マウント」** 動作が走ります (`ShellExecuteW` の `mount` 動詞)。マウント解除は通常通り、エクスプローラの右クリックメニュー「取り出し」から行ってください。

### 4.6 ユーザー コマンド (v1.13)

`%APPDATA%\fastfiler\commands\commands.json` に JSON 配列を書くだけで、右クリックメニューに任意の外部コマンドを追加できます。VSCode で開く / 7-Zip で圧縮 / ここで PowerShell — などを自分専用に整備可能。

**設置手順**

1. 設定 (`Ctrl+,`) → 基本タブ → 「ユーザー コマンド」の **📁 コマンド フォルダを開く**
2. 同フォルダの `commands.json.sample` を開いて中身をコピー
3. 同フォルダに `commands.json` を新規作成して貼り付け、編集
4. 設定タブの **🔄 再読込** ボタンで反映 (起動時にも自動ロード)

**スキーマ**

```jsonc
[
  {
    "id": "vscode",                 // 必須・一意
    "label": "VSCode で開く",       // 必須 (メニュー表示)
    "icon": "🆚",                    // 任意 (絵文字 1 文字)
    "exec": "code",                 // 必須 (PATH 解決可)
    "args": ["{path}"],             // 任意
    "cwd": "{parent}",              // 任意 (既定: 現在ペイン)
    "when": "any",                  // file|folder|any|background (既定 any)
    "extensions": [".md"],          // 任意 (when=file のとき AND 条件)
    "submenu": "開く",              // 任意 (グループ化サブメニュー)
    "shell": false,                 // true なら cmd /c 経由
    "hidden": false                 // true で一時無効化
  }
]
```

**プレースホルダ**

| 変数 | 展開 |
|---|---|
| `{path}` | 選択 1 件目のフルパス |
| `{paths}` | 全選択 (空白区切り、空白を含むものは自動でクオート) |
| `{name}` | basename (拡張子付) |
| `{stem}` | basename (拡張子なし) |
| `{ext}` | `.xxx` 形式の拡張子 |
| `{parent}` | 親フォルダ |
| `{cwd}` | 現在ペインのパス |
| `{count}` | 選択件数 |

**`when` (表示条件)**

| 値 | 表示タイミング |
|---|---|
| `file` | 選択がすべてファイル (1 件以上) |
| `folder` | 選択がすべてフォルダ |
| `any` (既定) | 1 件以上選択 (ファイル / フォルダ問わず) |
| `background` | 選択 0 件 (空白部の右クリック) |

`extensions` は `when=file` と組合わせて使い、選択全件が指定拡張子に一致するときのみ表示されます。

**実行例**

| 例 | 設定 |
|---|---|
| 7-Zip 圧縮 | `exec: "7z.exe"`, `args: ["a", "{parent}\\{stem}.7z", "{paths}"]`, `submenu: "圧縮"` |
| 7-Zip 展開 | `exec: "7z.exe"`, `args: ["x", "{path}", "-o{parent}", "-y"]`, `when: "file"`, `extensions: [".zip", ".7z"]` |
| ここで PowerShell | `exec: "powershell.exe"`, `args: ["-NoExit", "-Command", "Set-Location -LiteralPath '{cwd}'"]`, `when: "background"` |
| ImageMagick 縮小 | `exec: "magick"`, `args: ["{path}", "-resize", "50%", "{parent}\\{stem}_half{ext}"]`, `when: "file"`, `extensions: [".jpg", ".png"]` |

**注意**

- `commands.json` 本体には **コメント (`//`) を書けません** (標準 JSON 仕様)。サンプルのコメントは削除してください
- `exec` のフルパスにバックスラッシュを書く際は `\\` でエスケープ (JSON 文字列のため)
- `shell: true` のときは内部で `cmd /c "<exec> <args>"` に組み立て直すため、引数のクオートは Rust 側が自動で行います
- 実行は **非同期**。標準出力 / 標準エラーは現状取得していません (将来対応予定)
- 失敗時はダイアログで通知 (ファイル未存在など spawn エラー)

## 5. ナビゲーション

### 5.1 パンくず

ペインツールバーの中央が **パンくず** です。

- 各セグメントクリックでその階層へジャンプ
- セグメントへ **ファイルを D&D** で移動 (Ctrl でコピー)
- パスが長くて隠れる場合は **末尾 (現在フォルダ) が右側に残る** ように自動スクロールします
- 右端の `✎` (鉛筆) は常に表示。クリックでパス入力モードへ

### 5.2 アドレス入力 (Ctrl+L)

`Ctrl+L` または パンくず右端の `✎` でパス入力モードに切替。

- パス文字列を直接編集
- `Tab` で部分入力からの **フォルダ補完**
- `Esc` でパンくず表示に戻る
- 既存テキストは自動選択されるので、`Ctrl+C` で **そのままコピー** 可能

### 5.3 PC ビュー (ドライブ一覧)

ドライブ直下 (例: `C:\`) でさらに `Backspace` (または `↑` ボタン) を押すと **💻 PC ビュー** に遷移し、全ドライブをカード一覧表示します。クリックで該当ドライブへ移動。

### 5.4 ネットワークドライブ / UNC

#### ドライブ種別の認識

| 種別 | アイコン | 例 |
|---|---|---|
| 固定 (fixed) | 💽 | C:, D: |
| ネットワーク (network) | 🌐 | Z: → `\\server\share` |
| リムーバブル (removable) | 💾 | USB メモリ |
| CD/DVD (cdrom) | 💿 | 光学ドライブ |
| RAM ディスク (ram) | ⚡ | 仮想ドライブ |

`Z:` のようなマップ済みドライブにマウスホバーすると、対応する UNC パス (`\\server\share`) がツールチップで表示されます。

#### UNC パスでの直接アクセス

アドレスバーに `\\server\share[\sub...]` を入力して `Enter` で直接開けます。マップしていなくても利用可。

- パンくず分割: `\\server\share` を 1 ルートとして扱い、サブフォルダがチェーン化
- 親フォルダ移動: `\\server\share\sub` → `\\server\share` → ドライブ一覧
- ツリー: UNC ルートを起点に祖先自動展開

#### ツリーへの UNC 共有の自動登録

UNC を開くと `\\server\share` がワークスペースツリーの **🖥️ サーバノード** 配下に自動登録され、次回ワンクリックで再アクセスできます。

- サーバ単位 (`\\Himed`) でグルーピング → 配下に複数共有が並びます
- 共有ノードを **右クリック** で「ツリーから削除」可 (実体は削除されません)
- 登録は localStorage に保存され再起動後も保持

|---|---|

### 6.2 並べ替え

### 6.4 タブのロック (中クリック)

タブを **マウス中ボタン (ホイール) クリック** でロック / 解除。

- ロック中はタブの境界に **黄色のライン** が出ます
- `×` ボタンの位置に **🔒 アイコン** が表示され、クリックで解除可能
- ロック中タブは `Ctrl+W` や右上 `×` を押しても閉じません (誤クローズ防止)

### 6.5 タブのアイコン / ドライブ表示

タブ名の先頭にパス種別が表示されます。

| パス | 表示例 |
|---|---|
| ドライブ配下 (`C:\…`) | `C:` のドライブレターを文字で表示 |
| UNC (`\\server\share\…`) | 🌐 アイコン |
| その他 (PC ビュー等) | アイコンなし |

## 7. ペインとワークスペース

### 7.1 ペイン分割

各ペインのツールバーから:

- **┃ 縦分割** ── 左右に 2 ペイン
- **━ 横分割** ── 上下に 2 ペイン
- **× 閉じる** ── このペインを閉じる

中央の分割線をドラッグでサイズ調整。比率はタブと共に保存されます。

### 7.2 フォーカスペイン

直近で操作したペインが「フォーカスペイン」として記憶され、青い inset 枠で示されます。

- 切替タイミング: クリック / `Tab` / キー操作 / フォルダ移動
- タブ切替時は、そのタブで前回フォーカスしていたペインが自動復元
- ワークスペースツリーや「反映先=アクティブ」のジャンプ先になります

### 7.3 ワークスペースツリー

ドライブ起点のグローバルなツリーパネル。

- **`Ctrl+Shift+E`**: 表示 / 非表示
- ローカルドライブ (C:\, D:\, …) と登録済み UNC を複数ルートで並列表示
- フォルダクリックで「反映先」プルダウンの対象 (アクティブ / 🔴連動Red / 🔵連動Blue) に移動
- **キーボード操作**: パネルにフォーカスを当てて `↑/↓` で移動、`→` または `Enter` で展開、`←` で折畳または親へ
- **フォーカスペイン追従**: フォーカス中ペインの cd に追従して祖先を自動展開、ノードを画面内にスクロール
- パネル端をドラッグでサイズ変更

### 7.4 ペイン内ツリービュー

各ペインのツールバー 🌲 ボタンで **ツリー表示** に切替。`📋` でリストに戻ります。

ペイン内ツリーは **そのペインの path** に追従します (= 列挙対象を起点とした子ツリー)。

### 7.5 ドック式パネル配置

タブパネル / ワークスペースツリーパネルは `左 / 右 / 上 / 下 / 非表示` の 5 ヶ所に独立してドックできます。

#### パネルヘッダのドックボタン

| ボタン | 動作 |
|---|---|
| `◧` | 左にドック |
| `◨` | 右にドック |
| `⬒` | 上にドック |
| `⬓` | 下にドック |
| `✕` | 非表示 |

現在の dock 位置はボタンがハイライトされます。

#### サイズ変更

- 左 / 右 dock: パネルの内側端をドラッグで横サイズ
- 上 / 下 dock: パネルの内側端をドラッグで縦サイズ
- 140〜1200px の範囲で記憶、再起動後も保持

#### 同じ位置に複数パネルがある場合

設定 →「基本」→「ワークスペース配置」の **「同じ位置に複数あるとき 縦/横に積み重ねる」** で挙動を切替。

| 設定 | 挙動 |
|---|---|
| OFF (既定) | **並列表示** (3 列など) |
| ON | **積み重ね** (パネル間でサイズ分配) |

#### 関連ホットキー

- `Ctrl+B`: タブパネルの slot 循環 (左 → 右 → 非表示)
- `Ctrl+Shift+E`: ツリーパネルの表示 / 非表示

## 8. 検索

### 8.1 内蔵検索

`Ctrl+F` で検索パネルを開きます。

- **substring**: 含まれる文字列で検索 (既定、大文字小文字区別なし)
- **regex**: 正規表現
- ストリーミング表示で大量ヒットでも結果が即座に流れます
- 結果行をダブルクリックで該当フォルダへジャンプ
- もう一度 `Ctrl+F` で閉じる、`Esc` でも閉じる

### 8.2 Everything 連携

[Everything (voidtools)](https://www.voidtools.com/) の HTTP API と連携し、爆速検索が可能です。

#### セットアップ

1. Everything をインストールして起動
2. `Tools` → `Options` → `HTTP Server` を開き **Enable HTTP Server** をチェック (既定ポート 80)
3. FastFiler の `Ctrl+,` →「検索」タブ
4. **検索バックエンド** を `Everything HTTP` に変更、ポートを Everything と一致させる
5. `接続テスト` で ✅ なら OK

#### 動作

- 検索開始時に Everything HTTP API へクエリを投げて即時表示
- 失敗時は内蔵検索 (再帰列挙) に自動フォールバック (オレンジ表示)
- 「現在のフォルダ以下に限定」を ON にすると `path:"…"` で絞り込み

### 8.3 タブ単位の検索状態

- 検索パネルの **開閉状態 / クエリ / オプション** はペイン単位で保持
- タブ切替で:
  - 元タブの検索パネルはそのまま残る
  - 切替先で未起動なら表示されない
  - 元タブに戻るとクエリも復元

## 11. プラグイン

`%APPDATA%\fastfiler\plugins\<id>\` に `manifest.json` と `index.html` を含むフォルダを置くと、プラグインパネル (`Ctrl+Shift+P`) に一覧表示されます。

### 11.1 セキュリティと有効化

- 既定では **すべてのプラグインは無効**。各行のチェックボックスで明示的に有効化
- 有効化したプラグインのみ、行クリックでアクティブ化 (iframe 表示)
- API は `manifest.json` の `capabilities` で宣言されたものに限る

### 11.2 manifest.json

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "0.1.0",
  "description": "短い説明",
  "entry": "index.html",
  "capabilities": [
    "fs.read.dir", "fs.read.text", "fs.stat",
    "fs.write.text", "fs.mkdir", "fs.rename",
    "fs.copy", "fs.move", "fs.delete",
    "shell.open",
    "ui.notify", "ui.contextMenu.register",
    "pane.getActive", "pane.setPath",
    "storage.get", "storage.set"
  ]
}
```

### 11.3 SDK

`doc/plugins-sample/sdk.js` をプラグインフォルダの 1 階層上 (もしくは同階層) に置き、`<script src="../sdk.js"></script>` で読み込むと `window.ff` で型付き API が利用できます。

主な API:

| API | 説明 |
|---|---|
| `ff.fs.readDir` / `readText` / `writeText` / `mkdir` / `rename` / `copy` / `move` / `delete([paths], permanent?)` / `stat` | ファイル操作。`fs.delete` は既定でゴミ箱、`permanent:true` で完全削除 |
| `ff.shell.open(path)` | OS の既定アプリで開く |
| `ff.pane.getActive()` / `pane.setPath(path, paneId?)` | アクティブペインの取得 / 移動 |
| `ff.notify(message, "info"\|"warn"\|"error")` | 右下にトースト通知 |
| `ff.storage.get(key)` / `storage.set(key, value)` | プラグイン専用の永続 KV |
| `ff.registerContextMenuItem({ id, label, icon?, when?, extensions? })` | 右クリックメニューに項目追加 |

### 11.4 イベント (アプリ → プラグイン)

```js
ff.on("pane.changed",               e => { /* { paneId, path } */ });
ff.on("pane.selection.changed",     e => { /* { paneId, selection: string[] } */ });
ff.on("plugin.activated",           e => { /* {} */ });
ff.on("plugin.contextMenu.invoked", e => { /* { itemId, target: { path, name, isDir } } */ });
```

イベントは **アクティブ化中のプラグイン** にのみ届きます。コンテキストメニュー登録もアクティブのみ有効。

### 11.5 サンプル

| 場所 | 内容 |
|---|---|
| `doc/plugins-sample/hello-plugin/` | SDK / 読み書き / ペイン操作 / ストレージ / イベントログ |
| `doc/plugins-sample/context-menu-demo/` | 右クリックメニュー追加 (パス通知 / フォルダ列挙 / テキスト先頭表示) |

`doc/plugins-sample/sdk.js` も含めてフォルダごと `%APPDATA%\fastfiler\plugins\` 配下にコピーしてください。

### 11.6 プラグインパネルの操作

- パネル**左端をドラッグ**で幅変更 (220–900px、再起動後も保持)
- ヘッダ右上の `ctx=N` は登録中のコンテキストメニュー項目数 (デバッグ用)
- 📂 ボタンで `%APPDATA%\fastfiler\plugins\` を Explorer で開く
- ⟳ ボタンで一覧を再読込 (manifest 編集後など)
- 無効化すると登録中のコンテキストメニュー項目は自動削除、iframe は `about:blank`

### 11.7 内部仕様メモ

- iframe URL は Tauri の asset プロトコル (`http(s)://asset.localhost/...`)。`file://` は読めません
- `tauri.conf.json` の `app.security.assetProtocol` で `%APPDATA%\fastfiler\plugins\` 配下を許可
- 同時アクティブは 1 プラグイン (1 iframe)。切替時に前のプラグインの登録項目はクリア
- メッセージ形式: `{ __ff: "invoke", id, capability, args }` / `{ __ff: "result", id, ok, result?, error? }` / `{ __ff: "event", topic, payload }`

### 12.5 シェル統合 (フォルダ既定ハンドラ) — v1.12

「基本」タブ末尾の **「フォルダ既定ハンドラとして登録」** チェックを ON にすると、他アプリからフォルダパスを開く操作 (`ShellExecute("open", folder)`) が **FastFiler の新規タブ** で受け取れるようになります。

**動作**

- 既に FastFiler が起動中 → 新規タブとして指定フォルダが追加され、ウィンドウが前面化
- 起動していない → FastFiler が起動し、初期タブとして指定フォルダを開く
- 既に同じパスのタブがある → 重複追加せず、そのタブをアクティブ化
- コマンドラインからも `fastfiler.exe D:\work` で同様に開ける

**動作するケース (確認済)**

| ケース | 動作 |
|---|---|
| デスクトップでフォルダをダブルクリック | ✅ FastFiler |
| `Win+R` → `C:\path` → OK | ✅ FastFiler |
| コマンドプロンプトで `start "" "C:\path"` | ✅ FastFiler |
| ショートカット (.lnk) のフォルダリンク | ✅ FastFiler |

**動作しないケース (技術的制約)**

| ケース | 動作 | 理由 |
|---|---|---|
| **Excel ハイパーリンク** からフォルダを開く | ❌ Explorer | Excel は `IUniformResourceLocator` 経由で Explorer を直接呼び出しており、`Folder` クラスのレジストリ ハンドラを参照しないため |
| エクスプローラのアドレスバーで Enter | ❌ Explorer | Explorer 自身の操作のためシェル拡張をバイパス |

これらは Windows / Office の仕様上、HKCU レジストリでは介入できません (Files など他の代替ファイラでも同様の制約)。

**注意 (重要)**

- ON にすると **デスクトップやエクスプローラ内のフォルダ ダブルクリックも FastFiler が開きます**。Windows 側でアプリ別の区別はできません
- レジストリは `HKCU\Software\Classes\Folder\shell\open` と `Directory\shell\open` に書き込みます (HKCU スコープ・管理者権限不要)
- 内部的には `DelegateExecute` を空文字でオーバーライドして HKCR の標準 CLSID ハンドラを無効化しています
- OFF に戻せば Windows 標準のエクスプローラ動作に復帰します
- **FastFiler をアンインストール / 移動する前に必ず OFF にしてください** (登録 exe パスが残ったままだとフォルダが開けなくなります)
- 設定ダイアログの **🔍 診断** ボタンで現在のレジストリ値と exe パスを確認できます

## 14. 永続化されるデータ

すべて localStorage (キー: `fastfiler:state:v1`) または `%APPDATA%\fastfiler\` に保存され、再起動で復元されます。

- タブ構成、各タブの分割ツリー、分割比率、タブのロック状態
- 各ペインの現在パス、選択、スクロール位置、並べ替え条件、ペイン名
- 連動グループの設定 (4 チャネル ON/OFF)
- ホットキー割り当て
- 設定ダイアログの内容 (列数 / 隠しファイル / サムネイル / プラグインフォルダ / UI/ターミナルフォント)
- テーマ (system / light / dark)
- ワークスペース配置 (パネルの dock 位置 / サイズ / 並び順 / 積重ね設定)
- プラグインの有効/無効状態、プラグインパネル幅
- プラグインごとの KV ストア (`%APPDATA%\fastfiler\plugins\<id>\storage.json` に保存。localStorage ではない)
- 登録済み UNC 共有 (ワークスペースツリーの 🖥️ サーバノード)

## 15. トラブルシューティング

| 症状 | 対処 |
|---|---|
| ホットキーが効かない | 設定 → ホットキータブで重複していないか確認。IME ON 中は無視されます |
| サムネイルが出ない | 設定で ON / 対応形式 (png/jpg/gif/webp/bmp) か確認 |
| 大量ファイルでスクロールがガタつく | アニメーション拡張機能を OFF / Webview2 を最新に更新 |
| ペインが空白 | パスが存在しない or 権限不足。アドレスバー (`Ctrl+L`) に有効パスを入れて `Enter` |
| 設定をやり直したい | `localStorage.clear()` ですべて初期化 |
| 中ボタンクリックが効かない | OS 側でサイドボタン / ホイールクリックを別操作にバインドしていないか確認 |
| ターミナルフォントが反映されない | xterm.js の制約で **次回セッションから** 反映されます |
| UI フォント候補が少ない | `queryLocalFonts()` の許可ダイアログを許可する。拒否時は curated fallback のみ |
| プラグイン iframe が真っ白 / 「ff is not defined」 | `sdk.js` を `%APPDATA%\fastfiler\plugins\` 直下に置き、各プラグインから `../sdk.js` で参照しているか確認 |
| プラグインの右クリックメニューが出ない | プラグインを ✅ 有効化 + 行クリックでアクティブ化されているか確認 |
| シェル統合 ON 後、フォルダが開けなくなった (v1.12) | FastFiler.exe の場所が変わった可能性。設定 → シェル統合を一度 OFF → ON し直して再登録 |
| **Excel ハイパーリンクで FastFiler が開かない** (v1.12) | 仕様上の制約です。Excel は Explorer を直接呼ぶため HKCU レジストリ経由では介入できません。詳細は §12.5 の「動作しないケース」を参照 |
| FastFiler をアンインストールしたい (v1.12) | **必ず先に** 設定 → シェル統合を OFF にしてから削除してください (古い exe パスがレジストリに残るとフォルダが開けなくなります) |
