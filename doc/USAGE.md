# FastFiler 使い方ガイド

Windows 標準エクスプローラの代替として、**速度・縦タブ・ペイン連動・拡張性** を重視した自作ファイラです。本書では一通りの操作方法を順を追って説明します。

## 1. 起動と初回画面

開発モード:

```powershell
npm install
npm run tauri:dev
```

リリースビルド:

```powershell
npm run tauri:build
```

初回起動時はホームフォルダ（`C:\Users\<あなた>`）を 1 タブだけ開いた状態で立ち上がります。
2 回目以降は前回のセッション（タブ構成・パス・選択・スクロール位置・分割比率・ホットキー）を自動で復元します。

## 2. 画面構成

```
┌─────────────────────────────────────────────────────────────┐
│ ヘッダ ⚙ 設定                                               │
├──────────┬──────────────────────────────────────────────────┤
│          │ アドレスバー / ツールバー                        │
│ 縦タブ   ├──────────────────────────────────────────────────┤
│ ┌──┐ ┌──┐│                                                  │
│ │T1│ │T2││         ファイルリスト（仮想スクロール）         │
│ └──┘ └──┘│                                                  │
│   …      │                                                  │
│          ├──────────────────────────────────────────────────┤
│ ドライブ │ ステータスバー                                   │
└──────────┴──────────────────────────────────────────────────┘
```

- **左：縦タブペイン**　タブの一覧とドライブショートカット。列数を変更でき、ドラッグで並べ替え可能。
- **中央：作業領域**　タブごとに独立したペイン（ツリー）。さらに縦／横に分割できます。
- **上：ヘッダ**　歯車アイコンから設定ダイアログを開けます。
- **下：ステータスバー**　現在のフォルダの件数・選択数などを表示。

## 3. 基本操作

| やりたいこと | 操作 |
| --- | --- |
| フォルダを開く | ダブルクリック / `Enter` |
| 親フォルダへ | `Backspace` |
| 再読込 | `F5` |
| 複数選択 | `Ctrl+クリック` / `Shift+クリック` |
| 全選択 | `Ctrl+A` |
| アドレスバーへ移動 | アドレスバーをクリックしてパスを入力 → `Enter` |

ファイルリストは **行高 28px の仮想スクロール** で動作するので、`C:\Windows\System32` のような数万件フォルダでも瞬時に開きます。

## 4. 縦タブ

### 開く・閉じる

| 操作 | キー / UI |
| --- | --- |
| 新しいタブ | `Ctrl+T` または 縦タブ上部の「＋ 新規」ボタン |
| タブを閉じる | `Ctrl+W` または タブ右の `×` |
| タブ切替 | クリック |

### 並べ替え

タブを **ドラッグして並べ替え** できます。ドロップ位置に青いバーが出るので、その位置に挿入されます。

### 列数を変える

`Ctrl+,` で設定ダイアログを開き、「基本」タブの **タブ列数** で 1〜8 列を指定。アプリの再読込なしで即時反映されます。

### タブ名

タブ名は **現在開いているフォルダ名** が自動的に表示されます（フルパスはホバー時のツールチップで確認できます）。

## 5. ペイン分割と連動

### 分割

各ペインのツールバーから:

- **┃ 縦分割**　左右に 2 ペイン
- **━ 横分割**　上下に 2 ペイン
- **× 閉じる**　このペインを閉じる

中央の分割線をドラッグするとサイズを調整できます。比率はタブと共に保存されます。

### 連動（リンクグループ）

複数のペインを「同じグループ」に入れると、操作が連動します。デフォルトで **Red / Blue** の 2 グループが用意されています。

各ペインのツールバーから **Red / Blue / 連動なし** を選択。同じ色のペイン同士で以下が同期します。

| チャネル | 内容 |
| --- | --- |
| `path` | フォルダ移動を同期（ミラー閲覧に便利） |
| `selection` | 選択ファイルを同期 |
| `scroll` | スクロール位置を同期 |
| `sort` | 並べ替え条件を同期 |

`Ctrl+,` →「連動」タブで、**グループごとに 4 チャネルを ON/OFF** できます。
たとえば「Red は path だけ同期、selection は同期しない」といった細かい指定が可能です。

## 6. ファイル操作

| 操作 | デフォルトキー |
| --- | --- |
| コピー | `Ctrl+C` |
| 切り取り | `Ctrl+X` |
| 貼り付け | `Ctrl+V` |
| 名前の変更 | `F2` |
| 新規フォルダ | `Ctrl+Shift+N` |
| ゴミ箱へ送る | `Delete` |
| 完全削除（戻せません） | `Shift+Delete` |

`Delete` は OS のゴミ箱経由なので、誤操作時はゴミ箱から戻せます。
`Shift+Delete` は警告ダイアログを経て即時削除します。

## 7. ドラッグ＆ドロップ

- ファイルリスト → 別ペイン: **移動**（Ctrl 押しながらで **コピー**）
- ファイルリスト → 縦タブ: そのタブのフォルダへ移動 / コピー
- 外部アプリ → ファイルリスト: ファイルを取り込み

---

## 8. 検索

`Ctrl+F` で検索パネルを開きます。

- **substring**: 含まれる文字列で検索（既定、大文字小文字区別なし）
- **regex**: 正規表現
- ストリーミング表示なので大量ヒットでも結果が即座に流れてきます
- 結果行をダブルクリックすると、そのファイルのあるフォルダへジャンプ

---

## 9. プレビュー

`Ctrl+P` でプレビューパネルの表示／非表示を切り替えます。

| 種類 | 表示内容 |
| --- | --- |
| 画像 (png/jpg/gif/webp/bmp) | 等倍プレビュー |
| テキスト | 先頭 64KB を表示 |
| その他 | バイナリの先頭ダンプ |

---

## 10. サムネイル

設定ダイアログ「基本」で **サムネイル ON/OFF** を切り替え。
有効時、画像ファイルの行に小さなサムネイルが表示されます。
**画面に入った行のみ遅延ロード** されるので、大きなフォルダでも軽快に動きます。

---

## 11. 属性ダイアログ・コンテキストメニュー

ファイル/フォルダを **右クリック** すると、アプリ内コンテキストメニューが表示されます。

- 開く / 親フォルダ / 切り取り / コピー / 貼り付け / 名前の変更 / 削除
- **エクスプローラで表示**　Windows 標準のエクスプローラを起動
- **プロパティ**　サイズ・更新日時・属性を表示する属性ダイアログ

## 12. 設定ダイアログ

`Ctrl+,` または ヘッダの ⚙ アイコンから開きます。3 つのタブで構成されています。

### 基本

- **タブ列数**　縦タブの列数（1〜8）
- **隠しファイルを表示**　`.` 始まりや Hidden 属性の表示／非表示
- **サムネイル**　ON/OFF
- **プラグインフォルダ**　WebView プラグインを置くパス

### 連動

| グループ | path | selection | scroll | sort |
| --- | :-: | :-: | :-: | :-: |
| Red | ☐ | ☐ | ☐ | ☐ |
| Blue | ☐ | ☐ | ☐ | ☐ |

各セルをチェックすることで、対応するチャネルだけ同期させます。

### ホットキー

すべてのアクションに対して、自由なキー組み合わせを割り当てられます。

1. 一覧から変更したい行をクリック → セルが「キャプチャ中（青枠）」になります
2. 割り当てたいキーを **そのまま押下**（例: `Ctrl+Shift+R`）
3. 自動でセルに反映
4. `Esc` でキャンセル
5. 「初期値に戻す」ボタンで全アクション一括リセット

修飾キー単体（Ctrl だけ等）は受け付けません。必ず英数キー or `Enter` `Backspace` `F5` などと組み合わせてください。

## 14. プラグイン (v2.0 拡張)

`%APPDATA%\fastfiler\plugins\<id>\` に `manifest.json` と `index.html` を含むフォルダを置くと、
プラグインパネル (`Ctrl+Shift+P` または右上 🧩) に一覧表示されます。

### 14.1 セキュリティと有効化

- 既定では **すべてのプラグインは無効**。各行の左にあるチェックボックスで明示的に有効化してください。
- 有効化したプラグインのみ、行をクリックしてアクティブ化 (iframe 表示) できます。
- プラグインが呼び出せる API (capability) は `manifest.json` の `capabilities` で宣言されたものに限ります。

### 14.2 manifest.json

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "0.1.0",
  "description": "短い説明",
  "entry": "index.html",
  "capabilities": [
    "fs.read.dir", "fs.read.text", "fs.stat",
    "fs.write.text", "fs.mkdir", "fs.rename",
    "fs.copy", "fs.move", "fs.delete",
    "shell.open",
    "ui.notify", "ui.contextMenu.register",
    "pane.getActive", "pane.setPath",
    "storage.get", "storage.set"
  ]
}
```

### 14.3 SDK (推奨)

`doc/plugins-sample/sdk.js` をプラグインフォルダの 1 階層上、もしくは同階層に置き
`<script src="../sdk.js"></script>` で読み込むと `window.ff` 経由で型付き API を利用できます。

主な API:

| API | 説明 |
|---|---|
| `ff.fs.readDir(path)` / `readText` / `writeText` / `mkdir` / `rename` / `copy` / `move` / `delete([paths], permanent?)` / `stat` | ファイル操作。`fs.delete` は既定でゴミ箱、`permanent:true` で完全削除。 |
| `ff.shell.open(path)` | OS の既定アプリで開く |
| `ff.pane.getActive()` / `pane.setPath(path, paneId?)` | アクティブペインの取得 / 移動 |
| `ff.notify(message, "info"\|"warn"\|"error")` | アプリ右下にトースト通知を表示 |
| `ff.storage.get(key)` / `storage.set(key, value)` | プラグイン専用の永続 KV (`%APPDATA%\fastfiler\plugins\<id>\storage.json`) |
| `ff.registerContextMenuItem({ id, label, icon?, when?, extensions? })` | ファイル/フォルダ右クリックメニュー末尾に項目を追加 |

### 14.4 イベント (アプリ → プラグイン)

```js
ff.on("pane.changed",           e => { /* { paneId, path } */ });
ff.on("pane.selection.changed", e => { /* { paneId, selection: string[] } */ });
ff.on("plugin.activated",       e => { /* {} */ });
ff.on("plugin.contextMenu.invoked", e => { /* { itemId, target: { path, name, isDir } } */ });
```

注: イベントは「アクティブ化中のプラグイン」にのみ届きます。コンテキストメニュー登録もアクティブプラグインのみ有効です。

### 14.5 サンプル

| 場所 | 内容 |
|---|---|
| `doc/plugins-sample/hello-plugin/` | SDK / 読み書き / ペイン操作 / ストレージ / イベントログ |
| `doc/plugins-sample/context-menu-demo/` | コンテキストメニュー追加 (パス通知 / フォルダ列挙 / テキスト先頭表示) |

サンプルを使うには `doc/plugins-sample/sdk.js` も含めてフォルダごと
`%APPDATA%\fastfiler\plugins\` 配下にコピーしてください
(各プラグインから `../sdk.js` を参照する想定)。

### 14.6 プラグインパネル

- パネル**左端をドラッグ**して幅を変更可能 (220–900px、再起動後も保持)。
- ヘッダ右上の `ctx=N` は登録中のコンテキストメニュー項目数 (デバッグ用)。
- 📂 ボタンでプラグインフォルダ (`%APPDATA%\fastfiler\plugins\`) を Explorer で開く。
- ⟳ ボタンでプラグイン一覧を再読込 (manifest 編集後など)。
- プラグインを無効化すると、登録中のコンテキストメニュー項目は自動削除され、iframe は `about:blank` に戻ります。

### 14.7 内部仕様メモ

- iframe URL は Tauri の asset プロトコル (`http(s)://asset.localhost/...`) を使用。Tauri 2 のセキュリティ制約により `file://` は読めません。
- `tauri.conf.json` の `app.security.assetProtocol` で `%APPDATA%\fastfiler\plugins\` 配下を許可しています。
- 同時にアクティブにできるプラグインは 1 つ (1 iframe)。切替時に前のプラグインの登録項目はクリアされます。
- メッセージ形式: `{ __ff: "invoke", id, capability, args }` / `{ __ff: "result", id, ok, result?, error? }` / `{ __ff: "event", topic, payload }`。

## 15. 永続化されるデータ

すべて localStorage に保存され、再起動で復元されます。

- タブ構成、各タブの分割ツリー、分割比率
- 各ペインの現在パス、選択、スクロール位置、並べ替え条件
- 連動グループの設定（4 チャネル ON/OFF）
- ホットキー割り当て
- 設定ダイアログの内容（列数、隠しファイル、サムネイル、プラグインフォルダ）
- テーマ (system / light / dark)
- ワークスペース配置 (タブ位置、ツリー幅、タブ幅)
- プラグインの有効/無効状態、プラグインパネル幅
- プラグインごとの KV ストア (`%APPDATA%\fastfiler\plugins\<id>\storage.json` ファイル単位で保存。localStorage ではない)

リセットしたいときはアプリを終了し、Tauri のローカルデータフォルダ
`%APPDATA%\fastfiler\` 配下、または DevTools のコンソールで `localStorage.clear()` を実行してください。

## 16. トラブルシューティング

| 症状 | 対処 |
| --- | --- |
| ホットキーが効かない | 設定ダイアログ「ホットキー」で重複していないか確認。 IME ON 中は無視されます。 |
| サムネイルが出ない | 設定で ON になっているか / 対応形式（png/jpg/gif/webp/bmp）か確認。 |
| 大量ファイルでスクロールがガタつく | アニメーション拡張機能を OFF に。Webview2 を最新に更新。 |
| ペインが空白 | パスが存在しないか権限不足。アドレスバーに有効なパスを入れて `Enter`。 |
| 設定をやり直したい | `localStorage.clear()` ですべて初期化されます。 |
| プラグイン iframe が真っ白 / 「ff is not defined」 | `sdk.js` を `%APPDATA%\fastfiler\plugins\` 直下に置き、各プラグインから `../sdk.js` で参照しているか確認。 |
| プラグインの右クリックメニューが出ない | プラグインを ✅ 有効化＋プラグインパネルの行クリックでアクティブ化されているか確認。アクティブでないプラグインの登録は反映されません。 |

## 17. Everything 連携検索 (v1.1)

`Ctrl+F` の検索を爆速化したい場合、[Everything (voidtools)](https://www.voidtools.com/) を併用できます。

### セットアップ
1. Everything をインストールして起動。
2. `Tools` → `Options` → `HTTP Server` を開き、**Enable HTTP Server** をチェック。ポート (既定 80) を確認。
3. FastFiler の `Ctrl+,` で設定ダイアログ → `検索` タブ。
4. **検索バックエンド** を `Everything HTTP` に切替。ポートを Everything と一致させる。
5. `接続テスト` で ✅ が出れば OK。

### 動作
- 検索開始時に Everything HTTP API へクエリを投げ、結果を即座に表示。
- 失敗時は内蔵検索 (再帰列挙) に自動フォールバック (オレンジ表示)。
- 「現在のフォルダ以下に限定」を ON にすると `path:"…"` で絞り込み。

## 18. パンくず (v1.1)

アドレスバーは **パンくず** 表示になり、各セグメントクリックでその階層にジャンプ。
- セグメントへ **ファイル/フォルダをドラッグ&ドロップ** すると移動 (Ctrl でコピー)。
- `✎` ボタンでパス文字列を直接編集できる入力欄に切替。`Esc` で戻る。

## 19. ツリービュー (v1.1)

各ペインのツールバーの 🌲 ボタンで **ツリー表示** に切替。
- フォルダ階層を遅延展開 (`▸/▾`) で表示。
- ノードクリックでパスをそのペインに反映 (連動グループにも伝播)。
- `📋` ボタンでリスト表示に戻ります。
- 連動グループに入れた別ペインをリスト表示にしておくと、左ツリー / 右リストの古典スタイルが作れます。

## 20. タブ操作 (v1.2)

| キー | 動作 |
| --- | --- |
| `Ctrl+T` | 新しいタブ |
| `Ctrl+W` | 現在のタブを閉じる |
| `Ctrl+Tab` / `Ctrl+PageDown` | 次のタブへ |
| `Ctrl+Shift+Tab` / `Ctrl+PageUp` | 前のタブへ |
| `Ctrl+1`〜`Ctrl+8` | 指定インデックスのタブへ |
| `Ctrl+9` | 最後のタブへ |

`Ctrl+Tab` / `Ctrl+Shift+Tab` は設定 → ホットキータブで再割当可。

## 21. 検索ボックスの自動フォーカス & タブ単位 UI 状態 (v1.2)

- `Ctrl+F` を押すと検索パネルが開き、**入力欄に自動的にカーソル**が入ります（既存テキストは全選択）。
- 既に開いている状態で `Ctrl+F` を再度押すと、検索ボックスへ再フォーカスされます。
- `Esc` で検索パネルが閉じます。
- 検索パネルの **開閉状態 / 入力中のクエリ / オプション** はペイン単位で保持されます。タブを切替えても：
  - 元のタブで開いていた検索パネルはそのまま残る
  - 切替先のタブで未起動なら検索パネルは表示されない
  - 元のタブに戻ると、クエリも含めた状態がそのまま復元

## 22. タブの並べ替え (v1.3)

縦タブはマウスで掴んで順序変更できます。
- タブを **ドラッグ** すると半透明になり、ドロップ位置に青いインジケータが表示されます。
- ドロップで順序が確定し、再起動後も保持されます。

> **用語整理**
> - **タブ移動 (キーボード)** — `Ctrl+Tab` などでアクティブタブを切替
> - **タブの並べ替え** — マウスで掴んでタブの順序を変更（本節）
> - **タブ間ファイル移動 (D&D)** — ファイル行を別タブに落として移動 / コピー

## 23. ワークスペース配置 (v1.4)

UI を構成するパネル位置を切り替えられます。

### タブサイドバー位置
- **Ctrl+B**: 左 → 右 → 非表示 → 左 …の順に循環
- ヘッダーの 📑 ボタンでも同じ動作（アイコンが現状を示します: 📑 = 左 / 📑→ = 右 / 📑✕ = 非表示）
- 設定ダイアログ「基本」→「ワークスペース配置」でも明示的に選べます
- 幅は設定ダイアログから px 指定可能（既定 240）

### ツリーパネル
ワークスペース全体に対するツリービューで、ペイン内 TreeView (📋/🌲 切替) とは独立しています。
- **Ctrl+Shift+E**: 表示／非表示
- ヘッダーの 🌲 ボタンでも切替
- ローカルドライブ (C:\\, D:\\ …) を複数ルートとして並列表示
- フォルダクリックで「反映先」プルダウンの対象に移動:
  - **アクティブ**: 現在 **フォーカス中** のペイン (= 直近で操作した分割ペイン)
  - **🔴 連動Red**: 連動グループ Red に属する全ペインへ伝播
  - **🔵 連動Blue**: 連動グループ Blue に属する全ペインへ伝播
- **フォーカスペイン追従**: フォーカス中ペインのフォルダ移動に追従して自動的に祖先を展開し、現在ノードまでスクロールします。フォーカスは「ペインをクリック / キー操作 / アドレスバー編集」で切り替わり、青い枠で示されます。
- パネル右端をマウスでドラッグすると幅を変更できます（140〜600px）
- ▸/▾ で展開、ダブルクリックでも展開/折り畳み

## v1.5 追加機能

### 単一インスタンス
- 2 つ目を起動しようとしても既存ウィンドウがフォーカスされ、二重起動しません。

### テーマ切替（OS依存 / ダーク / ライト）
- 設定 → 「基本」タブ最上部のラジオで切替。`OS依存` は OS のライト/ダーク設定に追従します。

### ペイン名
- ペイン左上に 🏷 アイコン付きラベルが表示されます。**ダブルクリック**で編集モード、Enter 確定 / Escape キャンセル / 空欄で解除。

### ドライブ一覧（Backspace でドライブのトップへ）
- ドライブ直下 (例: `C:\`) でさらに `Backspace` (または ↑ ボタン) を押すと **💻 PC ビュー** に遷移し、全ドライブをカード一覧表示します。クリックで該当ドライブへ移動。

### Ctrl+F トグル
- 検索パネルが閉じている時: 開いて検索ボックスにフォーカス。
- 開いている時 (検索ボックス内含む): もう一度 `Ctrl+F` で閉じる。

### ツリー追従
- フォーカス中ペイン (青枠) のフォルダを移動すると、ワークスペースツリーが祖先パスを自動展開し、現在ノードを表示位置に収めます。
- フォーカスは「ペインのクリック・タブキー・キーボード操作」で切り替わります。タブを切替えた場合は、そのタブで前回操作していたペインが復元されます (無ければ代表 leaf)。
- 各ペイン内の 🌲 ツリービュー (📋/🌲 切替) は、自分自身の path に追従します。
